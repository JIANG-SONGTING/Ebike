7. Conclusion and Future Work

7.1 Conclusion

This project successfully designed and implemented a prototype IoT-based battery safety monitoring system for an electric motorcycle. Four out of five objectives were fully met through functional testing. Temperature alerts were confirmed at three different thresholds using hand warming and a hair dryer. ADC voltage measurement showed an average error of 1.0% against a multimeter. A sharp rotation of the MPU-6050 triggered a Large-angle rollover alert with GPS coordinates recorded. The alert processing speed was approximately 2 seconds from threshold exceedance to alert appearing in the app.

The contribution of this project is system integration. No existing commercial product or research prototype currently provides the same combination of battery temperature monitoring, voltage monitoring, motion data logging, GPS reporting, and 4G connectivity in one device designed for electric motorcycles. The GPS field test in Singapore's CBD also produced a reproducible finding that is relevant to any IoT system requiring reliable urban positioning. Single-constellation GPS is insufficient in dense high-rise urban environments, and multi-constellation GNSS is necessary.

7.2 Impact and Potential

The validated prototype provides the industry partner a functional foundation for further development towards a deployable product. The market gap identified in the literature review, namely an integrated safety module for electric motorcycles, remains unaddressed by any current commercial product. The system architecture is modular and can support additional sensors and fleet-scale deployment without fundamental redesign.

7.3 Future Work

The most immediate next step is to test voltage monitoring against a real lithium-ion battery under charge and discharge conditions, to confirm that the circuit and firmware logic work correctly when the input voltage changes over time. The voltage measurement circuit will also need to be redesigned for the 400V target battery with appropriate high-voltage isolation before the system can be deployed on the actual vehicle. The breadboard prototype should be converted to a PCB with vibration resistance and waterproofing to make it suitable for motorcycle mounting. A hardware cutoff circuit using a relay or MOSFET should be added so the system can physically disconnect the battery in response to a detected fault, rather than relying on notification alone. Multiple DS18B20 sensors should be added to monitor temperature at different points on the battery pack, which is straightforward because the 1-Wire protocol supports daisy-chaining multiple sensors on the same pin. A current sensor should be added to enable overcurrent detection and state-of-charge estimation. The gyroscope threshold should be calibrated using real riding data from the target motorcycle to ensure the Large-angle rollover alert can distinguish genuine crash events from normal cornering and road vibration. Finally, the completed system should be integrated and validated on the target vehicle under controlled conditions before road or track use.
