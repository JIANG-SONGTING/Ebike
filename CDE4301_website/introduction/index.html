1. Introduction

1.1 Background and Motivation

Electric motorcycles are becoming more popular as a cleaner alternative to petrol vehicles. They use lithium-ion battery packs as their main power source. Lithium-ion batteries can experience thermal runaway, a condition where heat builds up faster than it can escape, which can cause fire or explosion if triggered by physical damage, overcharging, or extreme temperatures (Hsieh et al., 2021). In Singapore, burn injuries from personal mobility device (PMD) fires increased 20 times between 2016 and 2020, and battery failure was found to be the cause in every case (Hsieh et al., 2021).

This project is done in collaboration with an industry partner developing a high-performance electric motorcycle for both road and track use. The vehicle uses a 400V lithium-ion battery pack. The company already has a prototype board with basic 4G communication and motion sensing, but it does not have any battery safety monitoring. There is no way to detect temperature or voltage anomalies, and no system to alert the rider remotely if something goes wrong. The risk is especially serious for motorcycles because there is no protective body around the rider, so a battery fire directly causes harm. The battery pack also faces more vibration and temperature variation compared to a car.

1.2 Project Objectives

This project aims to design and build a prototype IoT-based battery safety monitoring system with the following five objectives. The first objective is to measure battery temperature continuously and upload data to the cloud so that abnormal heat is detected before thermal runaway happens. The second is to measure battery voltage via an ADC resistor divider and upload to the cloud so that abnormal voltage levels can be identified. The third is to upload accelerometer, gyroscope, and GPS data to the cloud continuously so that impact events can be detected through threshold rules on the cloud platform. The fourth is to show current sensor readings on a local OLED screen so the rider can check status without needing a phone. The fifth is to send all sensor data to the OneNET cloud platform via 4G and display it in real time on the mobile app, with automatic email alerts when any value exceeds a threshold.

1.3 Scope and Boundaries

The prototype runs on USB power. For voltage monitoring, a resistor divider connected to the USB supply rail is used as a stable ADC input instead of a real battery. This validates circuit and firmware logic but does not test dynamic battery voltage behaviour. Working with the 400V battery pack is outside the safe scope of this project. Alert notifications use the OneNET personal tier, which supports email alerts. SMS alerts are available on the enterprise tier as a future upgrade. Items outside scope include dynamic battery testing, cell balancing, state-of-charge and state-of-health estimation, current monitoring, hardware cutoff protection, and high-voltage adaptation.

1.4 Novelty and Value Proposition

The market currently has three separate solution types for electric motorcycle safety. The first type is GPS trackers with basic voltage monitoring, such as Monimoto and SinoTrack. The second type is crash detection and SOS systems, such as GeoRide and CHIGEE AIO-6. The third type is traditional BMS solutions integrated inside battery packs. Each solves part of the problem but none provides a complete solution. GPS trackers do not monitor battery temperature. Crash detection systems do not check battery health. BMS solutions do not connect to the internet or report GPS location. The most advanced commercial products, such as the Teltonika FMC003, require OBD-II ports that most motorcycles do not have (Teltonika, 2024).

The contribution of this project is system integration. It brings existing sensing, communication, and cloud technologies into one unified safety monitoring system that no current product provides, designed specifically for electric motorcycles. The industry partner is a small startup company, so cost-effectiveness of components is a key consideration alongside performance. This project uses low-cost components including the STM32F103C8T6, AIR780EX, and OneNET platform, keeping the total bill of materials affordable and making further development practical for the partner. The system uses 4G instead of WiFi so it works on roads and racetracks where no WiFi access is available, and is designed for motorcycles where OBD-II solutions cannot be used.
